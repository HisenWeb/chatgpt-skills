# HTML 低保真原型生成触发 Prompt

请读取我提供的需求 / 页面规格 / 交接材料，并启用：

`skills/08-html-prototype-generator.md`

当前任务：
根据材料生成可本地打开的 HTML/CSS/JS 低保真原型。

执行要求：
1. 先识别材料用途，不要只按文件名判断，也不要假设旧窗口附件已经迁移。
2. 先判断本轮目标：全页面默认态、核心路径、指定页面、状态补充，还是既有原型修正。
3. 生成前必须确认输出形态：页面板模式，还是可点击原型模式。
   - 页面板模式：一个 HTML 中并排 / 分组展示多个页面或页面状态，适合截图评审。
   - 可点击原型模式：生成入口和页面跳转，适合验证流程和交互闭环。
   - 如果我没有明确说明，请先问我选择哪种模式。
4. 如果选择可点击原型模式，需要先确认轻量路由表；如果我没有提供，请先生成建议路由表让我确认。
   - 路由较复杂时，可以同时输出 Mermaid 流程图辅助审核。
   - Mermaid 只作为路由审核材料，不作为 HTML 原型运行依赖。
5. 如果材料冲突会影响页面范围、交互或功能边界，先指出冲突，不要自行合并后生成。
6. 只生成实际 HTML 原型，不生成图片参考图。
7. 只给可直接打开的原型预览入口，不生成压缩包。
8. 文件名、目录名、入口文件根据用户材料和页面语义决定，不套固定项目模板。
9. 默认使用原生 HTML/CSS/JS，不引入框架、不依赖后端、不需要安装依赖。
10. 示例数据集中管理，页面之间保持一致。
11. 不做真实登录、后端、云同步、在线代码运行等未确认扩展。
12. 生成预览后列出：
    - 原型预览入口；
    - 已覆盖内容；
    - 未覆盖 / 下一步；
    - 注意事项。
