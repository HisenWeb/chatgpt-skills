# HTML 低保真原型生成触发 Prompt

请读取我提供的需求 / 页面规格 / 交接材料，并启用：

`skills/08-html-prototype-generator.md`

当前任务：
根据材料生成可本地打开、可预览的 HTML/CSS/JS 低保真原型。

执行要求：
1. 先识别材料用途，不要只按文件名判断，也不要假设旧窗口附件已经迁移。
2. 先判断本轮目标：全页面默认态、核心路径、指定页面、状态补充，还是既有原型修正。
3. 生成前必须确认三项设置：
   - 目标端：手机端 / PC 端 / 两端都要 / 其他；
   - 输出形态：页面板模式，还是可点击原型模式；
   - 预览目标：ChatGPT 内嵌预览、本地浏览器打开，还是两者都要兼容。
4. 页面板模式：一个 HTML 中集中展示多个页面或页面状态，适合截图评审；页面较多时默认使用 Tab / 分组按钮直接切换，不用锚点滚动作为主要导航。
5. 可点击原型模式：生成入口和页面跳转，适合验证流程和交互闭环；如果我没有提供路由表，请先生成轻量路由表让我确认。路由较复杂时，可以同时输出 Mermaid 流程图辅助审核，但 Mermaid 不作为 HTML 运行依赖。
6. 弹窗、空状态、筛选态、确认态等页面状态默认归入所属页面内部，除非我明确要求拆成独立状态板。
7. 原型需要套预览容器：手机端使用手机容器，PC 端使用桌面 / 页面容器；长内容在容器内部滚动，不把外层页面撑高。
8. 如果预览目标包含 ChatGPT 内嵌预览，优先使用单 HTML、内联 CSS/JS、内部视图切换；如果是本地浏览器打开，多文件和相对路径可以使用，但要保证入口和主要链路可达。
9. 如果材料冲突会影响页面范围、交互或功能边界，先指出冲突，不要自行合并后生成。
10. 文件名、目录名、入口文件根据用户材料和页面语义决定，不套固定项目模板。
11. 默认使用原生 HTML/CSS/JS，不引入框架、不依赖后端、不需要安装依赖。
12. 示例数据集中管理，页面之间保持一致，文案尽量接近真实使用场景。
13. 生成后至少检查：入口可打开、首屏非空、CSS/JS 可用或已内联、主要 Tab/点击链路可用、容器不会被内容撑高。
14. 生成预览后列出：
    - 原型预览入口；
    - 已覆盖内容；
    - 未覆盖 / 下一步；
    - 注意事项。
