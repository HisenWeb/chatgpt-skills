# 模板：GitHub 项目调研启动与输出结构

适用场景：

- 已通过 `SKILL_INDEX.md` 启用 `skills/03-github-research.md`；
- 需要做 GitHub 项目调研前的需求澄清；
- 需要记录分类入口、搜索路径、候选分桶和项目证据卡；
- 需要输出正式 GitHub 调研结论或单个 repo 评估。

说明：

- 本模板是启动与输出结构，不是独立 Skill。
- 使用前应已启用 `skills/03-github-research.md`。
- 如果用户需求已经足够清楚，可以跳过启动确认卡，但必须声明默认假设。
- 不要把本模板当成固定长报告格式；简单问题可以裁剪。
- 本模板不得写死某一次调研的项目名、技术词、文件名、工具分桶或用户场景。
- 候选分桶、关键词和排除项必须根据当前任务动态生成。

## 调研启动确认卡

```text
调研启动确认卡

我先确认调研范围，避免直接按 stars 或单一关键词乱搜：

1. 你的主要目标是：
   - 找可直接使用的项目
   - 找同类替代方案
   - 判断某个 repo 是否靠谱
   - 比较多个 repo
   - 找更轻量方案
   - 找主流实现路线
   - 找特定生态里的成熟项目
   - 其他：...

2. 目标项目类型是：
   - 库
   - 框架
   - CLI
   - 插件 / 扩展
   - 模板 / starter
   - 应用
   - 服务
   - 平台
   - 数据集 / 规则集
   - 示例项目
   - awesome list / curated list
   - 其他：...

3. 运行环境 / 生态限制是：
   - 语言 / 技术栈：
   - 操作系统：
   - 本地 / 浏览器 / 编辑器 / 服务端：
   - 是否必须开源：
   - 是否接受商业服务：

4. 可接受的工具重量是：
   - 单文件 / 小库
   - 轻量 CLI
   - 本地应用
   - 插件 / 扩展
   - 服务端部署
   - 需要数据库 / 索引
   - 需要外部 API / 云账号
   - 团队级平台

5. 需要排除的是：
   - 不要停更
   - 不要重服务
   - 不要云依赖
   - 不要复杂部署
   - 不要 toy project
   - 不要 license 不清
   - 不要平台兼容风险明显的项目
   - 其他：...

6. 输出偏好是：
   - 只要 2~3 个推荐
   - 要完整候选清单
   - 要风险排序
   - 要单 repo 深度评估
   - 要后续实测方案

默认筛选方向建议：
- 轻量、本地、低侵入、易验证；
- 优先 GitHub 仓库证据；
- 外部网页只做佐证；
- 不按 stars 直接推荐；
- 不把上一次调研的分桶和关键词复用到本次任务。

请确认以上范围；如果直接开始，我将按默认假设继续。
```
