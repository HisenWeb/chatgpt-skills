# 模板：GitHub 项目调研启动与输出结构

适用场景：

- 已通过 `SKILL_INDEX.md` 启用 `skills/03-github-research.md`；
- 需要做 GitHub 项目调研前的需求澄清；
- 需要记录分类入口、搜索路径、候选分桶和项目证据卡；
- 需要输出正式 GitHub 调研结论或单个 repo 评估。

说明：

- 本模板是启动与输出结构，不是独立 Skill。
- 使用前应已启用 `skills/03-github-research.md`。
- 如果用户需求已经足够清楚，可以跳过启动确认卡，但必须声明默认假设。
- 不要把本模板当成固定长报告格式；简单问题可以裁剪。

## 调研启动确认卡

```text
调研启动确认卡

我先不直接搜项目，因为当前需求还需要收窄：

1. 你的主要目标是：
   - 快速理解老项目
   - 降低上下文 token
   - 生成代码图谱 / 架构图
   - 持续维护项目知识
   - 生成 CLAUDE.md / skills / AGENTS.md
   - 代码搜索 / 语义检索
   - 团队级 code intelligence
   - 其他：...

2. 项目类型是：
   - 前端 / 后端 / 全栈 / monorepo / 多语言 / 老框架 / 私有项目 / 其他：...

3. 你能接受的工具重量是：
   - 轻量 CLI，一次性生成文件
   - 本地可视化工具
   - Claude Code 插件 / Skill
   - MCP 服务
   - 向量数据库 / 索引服务
   - 团队级平台

4. 需要排除的是：
   - 不要 SaaS
   - 不要云服务
   - 不要向量库
   - 不要复杂部署
   - 不要侵入项目结构
   - 不要停更项目
   - 不要只适合 toy project 的工具

5. 输出偏好是：
   - 只要 2~3 个推荐
   - 要候选清单
   - 要风险排序
   - 要实测方案

默认筛选方向建议：
- 轻量、本地、低侵入；
- 优先 GitHub 仓库证据；
- 外部网页只做佐证；
- 不按 stars 直接推荐。

请确认以上范围；如果直接开始，我将按默认假设继续。
```
