# 模板：新窗口读取 Handoff 交接包 / 文件版

适用场景：

- `handoff-current.md` 已经由常规窗口或中转窗口生成；
- `handoff-evidence.md` 可能已经同时生成；
- 用户在新工作窗口上传交接材料，可能是 zip 交接包，也可能是若干单独文件；
- 可选：同一持续任务还配套上传了 `todolist.md`；
- 可选：交接材料中包含截图、日志、diff、源码包、聊天导出、测试结果或其他证据补充材料；
- 需要新窗口先识别交接材料清单，再读取核心 handoff 文件，并自动判断是否存在或应存在配套 TodoList 与 Evidence Pack。

复制到新工作窗口使用：

```text
我们继续上一个窗口的主题。我已上传新窗口交接材料，可能是 zip 包，也可能是若干单独文件。

请先识别交接材料清单，不要直接继续执行任务。

读取优先级：
1. 先找到并读取 handoff-current.md。
2. 如果存在 handoff-evidence.md，继续读取。
3. 如果存在 todolist.md，最后读取。
4. 如果存在证据补充材料，例如截图、日志、diff、源码包、聊天导出、测试结果或其他附件，请先列出它们的作用，不要假设已经理解其内容。
5. 如果 handoff-current.md 或 handoff-evidence.md 声明需要某些证据，但交接包里没有，请明确指出缺什么、为什么需要补充。
6. 如果 handoff-current.md、handoff-evidence.md、todolist.md 或证据补充材料之间存在冲突，先指出冲突，不要自行合并。

请先简短确认：
- 当前目标；
- 当前状态；
- 已确认结论；
- 必须遵守的规则 / 禁止事项；
- 证据缺口或待确认问题；
- 下一步建议。

要求：
- 不要重新发散。
- 不要重复旧窗口已完成的工作。
- 不要继承旧窗口中的错误尝试。
- 不要假设旧窗口附件、图片、zip、源码包已经自动迁移。
- 确认完成后再继续工作。
```
