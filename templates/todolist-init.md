# 模板：初始化极简 TodoList 主线清单

适用场景：

- 用户明确需要创建第一版 `todolist.md`；
- 用户要求建立主线 TodoList、大方向 TodoList 或防漂移 TodoList；
- 用户已经给出若干主线事项，并说明“把这些当成 TodoList”。

不适用场景：

- 只是一次性问答；
- 只是普通 handoff，且没有已启用的 `todolist.md`；
- 只是讨论长期方向，但还没有要求建立 TodoList；
- 只是维护 README、索引、doctor、模板等支撑设施，且用户没有把它们指定为主线事项。

重要规则：

- `todolist.md` 默认只使用 checkbox 清单。
- 不写“最终目标、当前阶段目标、当前 P0、支撑设施、验收 gate、下一步原子任务、候选项”等复杂结构。
- 如果用户已经给出清单，就按用户清单生成，不要重写成项目管理文档。
- 如果主线事项不清楚，只输出待确认候选清单，不生成正式 `todolist.md`。

复制到当前窗口使用：

```text
请启用 ChatGPT Skill：极简主线 TodoList 初始化。

我想为当前持续任务创建 todolist.md。请用极简 checkbox 清单，不要写复杂项目管理结构。

要求：
1. 先判断当前任务是否真的需要 todolist.md；如果只是一次性问答或临时任务，请直接说明不需要。
2. 如果我已经给出 TodoList 条目，必须以我的条目为准。
3. 正式 todolist.md 默认只能使用这种格式：

# TodoList

- [ ] 主线事项一
- [ ] 主线事项二
- [ ] 主线事项三

4. 不要写“最终目标、当前阶段目标、当前 P0、P1/P2、支撑设施、验收 gate、下一步原子任务、待确认候选、已废弃项”等复杂结构。
5. TodoList 条目只能是持续推进的主线事项，不要写聊天摘要、命令清单、文件修改清单、README / 索引 / doctor 等支撑设施。
6. 如果主线事项不清楚，只输出“待确认 TodoList”候选清单，让我确认；不要直接生成正式 todolist.md。
7. 输出要短，不要写成 handoff 或长篇规划。
```

如果用户已经明确给出条目，可使用：

```text
请启用 ChatGPT Skill：极简主线 TodoList 初始化。

下面这些就是正式 TodoList 主线项，请直接生成 todolist.md，不要添加其他结构：

- [ ] 事项一
- [ ] 事项二
- [ ] 事项三
```
