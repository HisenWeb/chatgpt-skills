# 模板：初始化 TodoList 主线推进账本

适用场景：

- 当前讨论已经从自由探索进入持续任务；
- 方向基本确定；
- 后续可能需要跨窗口继续推进；
- 需要创建第一版 `todolist.md` 来防止目标漂移。

复制到当前窗口使用：

```text
请启用 ChatGPT Skill：目标锚定与 TodoList 初始化。

请基于当前窗口已经确认的方向，生成第一版 todolist.md。

要求：
1. 先判断当前任务是否真的需要 todolist.md；如果只是一次性问答或临时任务，请直接说明不需要。
2. 生成前必须先做“目标锚定预检”，用两句话复述：最终目标、当前阶段目标。
3. 最终目标必须来自用户当前明确表达或当前窗口已确认方向；不能从 Skill 名称、模板名称、README、SKILL_INDEX、doctor、脚本、仓库维护上下文里反推。
4. todolist.md 是主线推进账本，不是聊天摘要、命令清单、文件修改清单、覆盖矩阵或仓库治理计划。
5. 必须区分：最终目标、当前阶段目标、支撑设施 / 验收 gate、大方向 TodoList、下一步原子任务、待确认 / 候选项、已完成 / 暂停 / 废弃。
6. 大方向 TodoList 必须围绕最终目标；不要把 doctor、README、SKILL_INDEX、模板、脚本维护杂项写成主线目标或 P0。
7. P0 必须是用户最终交付物或直接阻塞该交付物的当前阶段动作。
8. 下一步原子任务只能从当前 P0 或当前阶段目标中拆出一个最小动作，而且只能写一个。
9. 对证据不足或用户尚未确认的新方向，放入“待确认 / 候选项”，不要自动升级为 P0。
10. 如果发现目标锚定不清、上一版 TodoList 已经漂移，先指出源头问题，再生成修正版。
11. 优先生成可下载的 todolist.md 文件。
```
